# Security Policy

Thank you for helping keep BitCode projects safe and secure. We appreciate responsible security research and reports from the community.

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in a BitCode project, **please do not report it publicly**.

Instead:

* Do **not** create a public GitHub Issue.
* Do **not** post the vulnerability in GitHub Discussions.
* Contact the BitCode team privately with as much information as possible.

Please include:

* A description of the vulnerability
* Steps to reproduce the issue
* The affected project and version
* Proof of concept (if available)
* Potential impact
* Any suggested mitigation or fix (optional)

## Our Process

After receiving your report, we will:

1. Acknowledge receipt of the report.
2. Investigate and verify the issue.
3. Work on a fix if the report is valid.
4. Release a security update when appropriate.
5. Credit the reporter (with permission).

## Supported Versions

Security updates are generally provided for the latest stable release of each BitCode project. Older or unsupported versions may not receive security fixes.

## Responsible Disclosure

We ask that you allow us reasonable time to investigate and fix reported vulnerabilities before publicly disclosing them.

Responsible disclosure helps protect everyone using BitCode software.

## Thank You

Your efforts help make BitCode software safer for everyone. Thank you for reporting vulnerabilities responsibly.
